# Copy Variants

Read the profile's VOICE CASCADE first (project.md + profile.md + relevant
channel guidelines) — variants must stay on-voice and respect per-channel rules.

## Modes
1. **Cross-channel adaptation**: take a source post/brief and re-express it
   per target channel/platform. Not truncation — re-think the native format
   (X = punchy thread w/ standalone first tweet; LinkedIn = story+insight,
   line breaks; IG caption = hook line then depth; TikTok = spoken-word script).
   Honor each target channel's guidelines.
2. **Hook/headline batch**: 5-10 variants on one idea using distinct mechanisms
   (curiosity gap, bold claim, question, stat, story-open, contrarian) — labeled,
   so losing variants teach something.
3. **A/B pairs**: two versions differing on ONE variable, stated.

Rules: every variant respects banned words and CTA rules from the profile voice.
If the source idea is weak, say so before generating ten variants of weak.
