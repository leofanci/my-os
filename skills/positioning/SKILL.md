# Positioning

Read `projects/<project-slug>/strategy/intake.md` (+ assessment + competitor memo if present).

## Output: decision memo
1. **Category framing options** (2-3): position inside an existing category /
   subcategory / new category — pros/cons each (new category = expensive to teach).
2. **Differentiation claim** per option: the one thing competitors can't honestly say.
   Flag any claim the evidence log can't support.
3. **Messaging hierarchy** for the recommended option:
   - One-liner (≤15 words, customer language)
   - Elevator pitch (3 sentences: problem → solution → proof/why-now)
   - 3 supporting pillars, each with a proof point
4. **Test it**: 2 quick ways to validate (e.g., two one-liners as social bios or
   landing headlines, compare response).

Rules: customer's words from interviews > clever wordsmithing. No jargon the ICP
wouldn't use.
Save to `projects/<project-slug>/strategy/memos/positioning-vN.json`.
