# Launch Plan

Read `projects/<project-slug>/strategy/intake.md` + assessment (pace call) +
channel memo. If the pace call says validate_quietly, CHALLENGE the launch request
first — launching to validate is usually backwards. Proceed if the user confirms.

## Output
1. **Launch type options**: silent ship / soft launch to a niche / coordinated
   loud launch — pros/cons given stage and evidence. One recommendation.
2. **For the chosen type**:
   - Goal: ONE primary metric the launch must move + a number
   - Sequencing: pre-launch (asset list, seeding), day 0 (hour-by-hour if loud),
     week 1 (follow-up, capture learning)
   - Channel-specific actions only for channels from the channel memo
   - Asset checklist (page, posts, outreach messages) — note which the content
     skills can produce
3. **Failure plan**: what "flop" looks like and what you'll learn from it anyway.

Rules: a launch is an experiment too — define kill/success criteria like one.
No "go viral" plans.
Save to `projects/<project-slug>/strategy/memos/launch-vN.json`.
