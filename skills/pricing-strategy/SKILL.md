# Pricing Strategy

Read `projects/<project-slug>/strategy/intake.md` + competitor memo if present.

## Output: decision memo
1. **Model options** (2-3 of): one-time, subscription, usage-based, freemium,
   free+services — pros/cons FOR THIS project (stage, ICP, sales motion).
2. **Price point reasoning**: anchor to value delivered and competitor range,
   not cost. Give a number or narrow range per option; state assumptions.
3. **Packaging**: if tiers, max 3; what's the "most people should pick this" tier.
4. **Stage warning**: pre-validation, pricing is a discovery tool — recommend a
   price TEST (e.g., presale at target price) over a pricing committee. Charging
   something ≫ free signups as evidence.
5. One recommendation + what evidence would change it.

Rules: founders underprice by default — push against it explicitly. Freemium is a
growth strategy with real costs, not a safety blanket; say when it's wrong.
Save to `projects/<project-slug>/strategy/memos/pricing-vN.json`.
